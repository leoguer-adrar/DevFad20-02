		

						Contexte du projet :

	C'est pour un restaurateur qui aimerais la refonte de son site fait actuellement avec un CMS. 	

	L'application sera en 3 parties sur des serveurs differents et communiquerons par une API:	
		-> Une application cliente pour le  site web : Site du restaurant (6 pages) 
			Les pages/onglets necessaires :
				- Acceuil
				- Le restaurant
				- le Chef
				- La Carte
				- Infos Pratiques
				- Contact
				- Reserver

		-> Une application cliente de gestion pour l'admin: Un tableau de bord pour la gestion de contenu et relation client(message) (1 page)
			Les onglets necessaires :
				- Profil
				- Reservations
				- Messages 
				- Menus			

		-> une API : Sert d'intermediaire pour les applications et la BDD

	Liste des fonctionnalitées :
		-> App Site Web :
			- Affichage des differentes parties du site 
			- Envoie de formulaire de contact 
			- Demande de reservation 
			- Site traduit dans differents languages 		
		
		-> App gestion Admin
			- Connexion/deconnexion
			- Lecture/Ajout/Modification sur les reservations
			- Lecture/modification sur les messages
			- Lecture/Modification des Menus
		
		-> API
			- Revevoir/repondre des requetes aux applications clientes 
			- operations sur la BDD
